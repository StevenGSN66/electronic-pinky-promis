# 亲子信任约定小程序 — 需求记录与提示词

## 项目概述
开发一款面向亲子关系的"电子拉钩"信任约定小程序，通过游戏化的方式帮助孩子和家长建立、记录和维护彼此的约定，培养守信意识。

---

## 一、核心功能需求

### 1. 用户引导与档案
- **首次引导页**（仅首次展示）：三步引导流程
  - 第一步：选择喜欢的头像
  - 第二步：填写昵称
  - 第三步：添加常用约定人（如妈妈、爸爸等）
- **持久化机制**：使用 `localStorage` 存储 `hasSetAvatar` 标记，完成后不再显示引导页
- **个人页面**：可修改头像、查看守信勋章、注销账户
- **注销账户**：清除所有本地数据，返回引导页重新设置

### 2. 约定管理
- **新建约定**：
  - 必填：选择约定双方（发起承诺人 + 接受承诺人）
  - 双方从常用约定人中选择，不能是同一人
  - 填写约定内容，AI智能解析（约定事项、截止时间、奖惩标准）
- **约定类型自动推断**：根据发起人/接受人关系自动判定
- **签名确认**：双方拉钩签名，生成约定卡片
- **履约打卡**：完成约定后点击"已完成"确认
- **逾期复盘**：逾期后可进行AI复盘

### 3. 信任树可视化
- **信任树首页**：以树形可视化展示所有约定
  - 树上挂有叶子形状的约定卡片
  - 已兑现的叶子显示绿色，待复盘显示黄色
  - 信任值评分系统
  - 本周约定进度（环形图 + 星期标记）

### 4. 勋章系统
- **守信勋章**：3x3表格形式展示
  - 达成条件时勋章"亮灯"（绿色边框 + 渐变背景）
  - 未达成时"不亮灯"（低透明度 + 灰度滤镜）
- 勋章包括：守信新星、履约达人、信任守护者等

### 5. 提醒功能
- **到期温和提醒**：约定截止前弹出友好提醒弹窗
- **连续打卡提醒**：完成约定后的鼓励提示

## 二、页面结构

| 页面 | 功能描述 |
|------|---------|
| 首页 | 主操作页面：今日约定、进行中的约定列表、已完成/逾期列表、守信勋章、新建约定入口 |
| 信任树 | 可视化概览：信任树图形、约定叶子卡片、AI复盘、本周进度 |
| 我的 | 个人档案：头像、昵称、守信称号、账户设置、注销账户 |
| 新建约定 | 创建新约定：选择双方、AI解析、生成约定 |
| 签名确认 | 双方签名：拉钩动画、签名canvas、生成约定卡片 |
| 约定复盘 | 逾期复盘：填写原因、AI鼓励、更新信任树 |

## 四、技术栈

- **前端**：纯 HTML + CSS + JavaScript（单文件原型）
- **存储**：localStorage + sessionStorage（多重保障）
- **响应式**：移动端优先 + 桌面端侧边栏适配
- **图标**：Emoji 为主
- **动画**：纯 CSS 动画

## 五、提示词记录（按开发顺序）

### 初始需求
> 帮我做一个亲子信任约定小程序的原型，包含信任树、新建约定、签名确认、到期提醒、复盘等功能。

### 引导页需求
> 如果你之前没有见过自己的个人档案，才需要新建头像、名称以及常用约定人。如果你之前已经建过了，则无需每次登录都重新创建。

### 约定双方选择
> 名称不是常用约定人，所以新建约定后，发起承诺人和接受承诺人都不能是名称。
> 发起承诺人和接受承诺人不能是"请选择接受承诺人"，该选项不可用。

### 签名确认修复
> 电子拉钩的签名确认环节中，完成第一个约定后，后面的所有约定到签名确认环节都不会再显示签名确认窗口。

### 引导页重复弹出修复
> 登录完成、刷新页面、重新进入应用时，不再重复弹出「新建头像/首次引导页面」，仅第一次打开才展示。
> 技术实现：使用 localStorage 持久化存储标识 hasSetAvatar。

### 布局重构
> 修改底部导航栏：最左侧为首页，中间改为信任树，最右侧保持"我的"不变。将信任树页面改成图一的样子。

### 勋章与新建约定
> 守信勋章做成表格形式，达成条件时亮灯，未达成不亮灯。首页守信勋章下新增"新建约定"入口栏。

### 注销账户
> 在头像和名称区域添加点击入口，下方增加注销账户选项。点击后弹出确认弹窗。
