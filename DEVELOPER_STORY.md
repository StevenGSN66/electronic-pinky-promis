# 我和 TRAE 一起造出"信任星球"的真实故事

## 一、脑子里有个想法，但完全不会写代码

我是五年级的小学生。平时跟爸妈口头约定的"拉钩"，经常因为各种原因泡汤——大人忘了，或者我也耍赖。我就想：要是有一个"电子拉钩"的小程序，把约定记录下来，到期提醒，违约还能复盘，那该多好啊！

但是我完全不会写代码。html 是什么？css 是什么？javascript 是什么？我只知道它们大概是做网页用的。Python 我稍微接触过一点，但也只会 print("hello world") 这种最基础的。

然后我听说了 TRAE——一个你只要用说话的方式告诉它你想做什么，它就能帮你把网页做出来的 AI 工具。我就决定试试。

## 二、第一步：把"设计图"喂给 TRAE

我手头只有一张 PDF 的设计参考图，上面画了一个信任树的界面。我直接把它上传到 TRAE，然后对它说：

> "我上传了这张图，帮我做一个这样的网页，要包含信任树、新建约定、签名确认、复盘这些功能。"

TRAE 看了一眼图，就开始生成代码了。几分钟之后，一个完整的 HTML 文件出来了——有首页、有创建页面、有签名页面、有复盘页面，甚至还有导航栏和路由切换！

我当时就震惊了：**原来真的只要说话就行啊！**

## 三、最神奇的时刻：从"设计图"到"能点能交互"的网页

那一步让我觉得"原来这么简单"的，是 TRAE 帮我把一张静态的设计参考图，变成了一个真的能点、能填、能保存数据的网页。

我以前以为做网页需要学很多很多东西：html 标签、css 样式、js 逻辑、数据库、服务器……结果 TRAE 一次性就帮我生成了一个完整的单文件 HTML，所有的交互逻辑、页面切换、数据存储（用的是浏览器的 localStorage）全都写好了。

我只需要在浏览器里打开这个文件，就能看到一个有 5 个页面、能新建约定、能签名、能查看信任树的完整应用。

那一刻我真的觉得：**原来编程没有那么高不可攀，只要你能把想法说清楚，TRAE 就能帮你实现。**

## 四、那些我以为搞不定的坎，TRAE 帮我跨过去了

### 坎 1：信任树的叶子要覆盖在树的位置上

设计图里，信任树的每片叶子上要写约定的内容。我一开始完全不知道怎么让叶子"贴"在树的正确位置上。TRAE 告诉我可以用 CSS 的绝对定位，根据树图片的坐标来放置叶子，还自动帮我生成了计算位置的代码。

> 我说："叶子的位置好像不太对。"
> TRAE 说："叶子目前是靠左排列的，我来改成按百分比位置覆盖到树上。"

然后它自动调整了 CSS，叶子就准确地出现在树上了。

### 坎 2：我想加一个吉祥物"雁小约"，还有视频引导

我有一些雁小约的图片和视频素材，想让它们出现在网页里。但我不知道图片要去掉背景才能好看，也不知道视频要转成什么格式才能在网页上播放。

TRAE 直接帮我一站式解决了：
- 用 Python 的 PIL 库自动去除图片的白色背景，变成透明 PNG
- 用 ffmpeg 把视频转成 WebM 格式（支持透明背景）
- 还帮我写了视频播放的控制逻辑：自动播放一次，45 秒无操作再重新播放

这些都是我完全不会的技术，但 TRAE 就像一个有经验的程序员搭档，我说"我想要这个效果"，它就能从技术层面帮我实现。

### 坎 3：DeepSeek AI 接入——让 AI 自动解析约定

我希望输入一段口语化的约定文字，比如"妈妈说如果我每天练琴30分钟，周末就带我去公园"，然后系统自动提取出：
- 承诺人：妈妈
- 履约人：我
- 条件：每天练琴30分钟
- 奖励：周末去公园

这需要调用 AI API，还要写 Prompt。我完全不懂 API 怎么调、Prompt 怎么写。

TRAE 直接帮我：
- 在代码里接入了 DeepSeek API
- 写了完整的解析 Prompt
- 还加了错误处理和加载状态

我只说了一句"能不能让 AI 自动解析约定内容"，TRAE 就帮我搞定了。

### 坎 4：GitHub Pages 部署——让所有人都能访问

最后一步是把网页发布到网上，让其他人也能打开。我听说过 GitHub，但完全不知道怎么用。

TRAE 一步一步教我：
- 初始化 git 仓库
- 提交代码
- 推送到 GitHub
- 设置 gh-pages 分支
- 告诉我在哪能找到在线链接

现在，我的"信任星球"已经有一个公开的网址了：https://stevengsn66.github.io/electronic-pinky-promis/

## 五、我的真实感受

### 1. TRAE 不是替你思考，而是帮你把想法落地

很多人以为用 AI 做开发就是"让它自己想"。其实不是。TRAE 更像是一个"超级翻译官"——你把脑子里的想法用大白话讲给它听，它帮你翻译成代码。

比如我说："这个按钮太小了，能不能大一点？"它就会去找对应的 CSS 代码，把 width 和 height 调大。我说："这个页面的背景色我想要淡绿色的。"它就会去找 background-color 的属性，改成 #e8f5e9。

**关键是：你得先知道你想要什么。** TRAE 不会替你决定产品长什么样，但它能帮你把你描述的样子做出来。

### 2. 反复沟通是常态，别怕"麻烦"它

开发过程中，我经常是：
- "这个图标能不能换成我上传的新图片？"
- "按钮的位置往右移一点。"
- "视频没有声音，检查一下。"
- "这个页面打开后没有内容，是不是代码出错了？"

TRAE 每次都会耐心地分析、修改、再测试。有时候一个问题要来回三四次才能完全解决，但它从来不会不耐烦。

这让我明白：**用 AI 开发不是"一键生成"，而是一个"迭代打磨"的过程。** 你提需求，它实现，你测试，你再提修改意见，它再改。循环往复，直到满意。

### 3. 最意外的收获：我顺便"偷学"了很多代码知识

虽然我不会写代码，但看了 TRAE 生成的那么多代码之后，我现在大概知道：
- HTML 是网页的骨架（有什么元素）
- CSS 是网页的皮肤（长什么样）
- JavaScript 是网页的大脑（能做什么交互）
- localStorage 是浏览器的"小仓库"（能存数据）
- git 是代码的"时光机"（能保存不同版本）

TRAE 每次修改代码时，都会告诉我它改了什么、为什么这样改。不知不觉间，我也学到了很多编程的概念。

我只是一个五年级的小学生，连 html 标签都不认识几个。但靠着 TRAE，我做出了一个完整的、有 5 个页面、有 AI 解析、有语音输入、有视频引导、能签名、能复盘、能成长的"信任星球"应用。

这感觉就像——你脑子里有一个城堡的蓝图，但你不会砌砖。TRAE 就是那个能看懂你的蓝图，并且一块砖一块砖帮你把城堡建起来的神奇工匠。

**感谢 TRAE，让我这个"编程小白"也能把脑子里的想法变成真正能用的小程序。**
